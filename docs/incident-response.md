# Incident Response

## Severity
- Sev-1: Client-facing outage or data risk
- Sev-2: Partial degradation or security alert
- Sev-3: Minor defects and performance issues

## Triage Checklist
- Define impact, start timer, assign incident commander
- Gather logs/metrics (CloudWatch), recent deployments
- Mitigate (rollback, scale, disable feature flag)
- Document root cause, corrective actions, and follow-ups

## Communications
- Internal channel and stakeholder updates every N minutes
- Post-incident review within 48 hours
