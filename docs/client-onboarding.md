# Client Onboarding

Welcome! This guide explains how we collaborate to deliver your AWS DevOps platform.

## Prerequisites
- AWS account with administrator access (or scoped roles)
- GitHub repository (or we can host)
- Domain and SSL preferences

## Steps
1. Discovery: goals, constraints, security/compliance needs
2. Architecture: finalize blueprint and environments
3. Provisioning: Terraform VPC, networking, and platform services
4. CI/CD: enable pipelines, ECR, and deployments
5. Security: Trivy and Checkov gates; IAM and SG hardening
6. Observability: CloudWatch dashboards/alarms
7. Knowledge transfer: runbooks and handover

## Access & Secrets
- Secrets stored in GitHub Actions secrets or AWS Secrets Manager
- Least privilege IAM roles and IRSA for EKS
